# Персоны — CrowdProj Marketplace
- **Primary persona:** Закупщик (Инженер по закупкам) — платящий сегмент, принимает решение о выборе платформы
- **Secondary personas:** Поставщик (производитель комплектующих), Менеджер по продажам (сбыт предприятия) — создают ликвидность и наполнение платформы
## Персона 1: Закупщик (Инженер по закупкам)
| Атрибут                     | Описание                                                                                 |
|-----------------------------|------------------------------------------------------------------------------------------|
| **Возраст**                 | 30–50 лет                                                                                |
| **Род занятий**             | Инженер по закупкам / Главный инженер на промышленном предприятии (100–5 000 чел.)       |
| **Цели**                    | • Найти надёжных поставщиков комплектующих по техпараметрам (ТЗ/спецификация) в одном окне<br>• Диверсифицировать цепочку поставок, снизить риск остановки производства<br>• Сократить время поиска альтернатив с 1–3 недель до 2 дней<br>• Наладить регулярные B2B-поставки с автозаявками и электронным документооборотом |
| **Боли**                    | • Низкая прозрачность рынка — поиск альтернативных поставщиков по ТЗ занимает недели ручного сбора данных (Google, Avito, Пульс цен, личные связи) — **серьёзность 9/10**<br>• Жёсткая зависимость от 2–3 проверенных поставщиков — срыв одного блокирует производство **(серьёзность 9/10)**<br>• Разрозненность данных — приходится работать через множество источников<br>• Страх подвести руководство при срыве поставок<br>• Нет инструментов для повторяющихся закупок — каждый заказ оформляется вручную, нет автозаявок по графику **(серьёзность 7/10)** |
| **Техническая грамотность** | Средняя (Excel, 1С, CRM, работа с технической документацией)                             |
### Сценарий использования
На предприятии срочно потребовалась партия фланцев определённого ГОСТа. Основной поставщик задерживает отгрузку на 3 недели. Инженер по закупкам заходит на CrowdProj Marketplace, создаёт заявку с прикреплённым ТЗ и спецификацией. Система подбирает подходящих поставщиков. Закупщик видит 4 верифицированные компании, сравнивает цены, сроки и отзывы, отправляет запрос коммерческих предложений. В течение дня получает 3 варианта, выбирает оптимальный — поставка через 5 дней.
### Цитата
> «Я трачу недели на поиск нормального поставщика. Если бы можно было за пару дней найти проверенную альтернативу — это сняло бы огромную головную боль и спасло бы производство от простоев.»

## Персона 2: Поставщик (производитель комплектующих)
| Атрибут                     | Описание                                                                                   |
|-----------------------------|--------------------------------------------------------------------------------------------|
| **Возраст**                 | 35–55 лет                                                                                  |
| **Род занятий**             | Владелец / Коммерческий директор малой или средней производственной компании (10–50 чел.)   |
| **Цели**                    | • Найти новые заказы и долгосрочные контракты с промышленными предприятиями<br>• Узнать, какие комплектующие сейчас в дефиците на рынке (аналитика спроса)<br>• Повысить загрузку производства и обеспечить регулярный поток заказов |
| **Боли**                    | • Нет видимости реального спроса — компания не знает, какие комплектующие сейчас нужны заказчикам **(серьёзность 8/10)**<br>• Информация о заказах поступает только через личные связи — фрагментарно и несистемно **(серьёзность 7/10)**<br>• Сложно «пробиться» к крупным заказчикам без наработанных контактов **(серьёзность 7/10)**<br>• Потеря времени на нецелевые тендеры и холодные звонки<br>• Нет понимания конкурентной ситуации и цен по категориям |
| **Техническая грамотность** | Средняя (Excel, CRM, 1С, базовое понимание технической документации)                       |
### Сценарий использования
Владелец производственной компании по выпуску промышленных комплектующих утром открывает CrowdProj Marketplace, просматривает ленту новых заявок от заводов. Фильтрует по своему профилю (токарные работы, фланцы, регион). Видит заявку от машиностроительного завода на партию крепёжных изделий. Изучает детали: объём, сроки, бюджет. Откликается на заявку, прикладывает коммерческое предложение и сертификаты продукции. В течение 2 дней получает подтверждение и приглашение на подписание контракта.
### Цитата
> «Мы можем делать качественные комплектующие, но о нас никто не знает. Хотелось бы видеть, где мы реально нужны, а не тратить время на обзвон знакомых и холодные продажи.»

## Персона 3: Менеджер по продажам (сбыт промышленного предприятия)
| Атрибут                     | Описание                                                                                   |
|-----------------------------|--------------------------------------------------------------------------------------------|
| **Возраст**                 | 30–45 лет                                                                                  |
| **Род занятий**             | Менеджер по продажам / Менеджер по сбыту на промышленном предприятии (100–5 000 чел.)       |
| **Цели**                    | • Получать целевые заявки от закупщиков на продукцию предприятия — ≥5 в месяц<br>• Видеть реальный спрос на рынке — какие комплектующие и услуги сейчас в дефиците<br>• Оперативно реагировать на запросы без ручного поиска и обзвонов<br>• Увеличить загрузку производственных мощностей предприятия |
| **Боли**                    | • Нет информации о реальном спросе на рынке — менеджер не знает, какие комплектующие сейчас нужны заказчикам **(серьёзность 8/10)**<br>• Трата времени на обзвон знакомых закупщиков для разведки спроса<br>• Потеря заказов из-за медленной реакции — информация приходит через личные связи, а не системно<br>• Нет инструмента для автоматического подбора продукции под заявки по техпараметрам |
| **Техническая грамотность** | Средняя (CRM, Excel, 1С, работа с технической документацией)                               |
### Сценарий использования
Менеджер по продажам на промышленном предприятии зарегистрировал продукцию компании на CrowdProj Marketplace — загрузил карточки товаров с техпараметрами, ГОСТами, ценами и сертификатами. Система AI сама сопоставляет загруженную номенклатуру с входящими заявками от закупщиков. Менеджер получает уведомление: «По вашей позиции „Фланец ГОСТ 12820-80“ поступил запрос от завода им. Ленина». Открывает детали заявки, смотрит объём и сроки, подтверждает готовность к поставке. Без ручного поиска и обзвонов — AI сделал матчинг за него.
### Цитата
> «Я трачу время на обзвон знакомых закупщиков, чтобы узнать, что им нужно. Если бы AI сам подсказывал, на какие мои позиции есть спрос — это сэкономило бы часы в день.»
