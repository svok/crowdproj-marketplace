## Capability Expectations
| Сценарий (ID) | Ожидаемая возможность | Комментарий |
|---------------|-----------------------|-------------|
| US-001, US-008 | Регистрация предприятия (профиль: название, ИНН, контакты, сфера деятельности, регион) | Без регистрации невозможен запуск two-sided marketplace — premortem Risk 1 (chicken-and-egg) |
| US-018 | Приглашение сотрудника по email + ролевая модель (администратор/сотрудник) | Многопользовательский режим внутри одного предприятия из Personas (3 роли работают от имени одной компании) |
| US-019 | Принятие приглашения + установка пароля + аутентификация | Вход через ссылку-приглашение без повторной регистрации предприятия |
| US-002, US-009 | Создание объявления с direction (sell/buy), категорией и динамическими EAV-атрибутами | Единая сущность объявления с двумя направлениями — ключевой механизм наполнения платформы контентом |
| US-003, US-010 | Базовый матчинг: показ встречных объявлений при создании/обновлении (полнотекстовый поиск + фильтры) | Ценность платформы: закупщик видит поставщиков, поставщик — спрос. Без этого доска объявлений без автоматизации |
| US-004, US-012 | Реестр предприятий с полнотекстовым поиском и фильтром по региону | Митигация chicken-and-egg: видимость активности даже при малом числе объявлений |
| US-005, US-015 | Список объявлений предприятия (активные) + просмотр из карточки предприятия | Оценка ассортимента поставщика без отдельного запроса |
| US-006, US-011 | Глобальный поиск по всем объявлениям (название, категория, direction, регион) | Поиск товаров/заявок без ожидания матчинга при создании объявления |
| US-007, US-013 | Email-напоминалка: одно письмо при неактивности дольше настроенного периода и наличии новых матчингов | Удержание пользователей — возврат на платформу, снижение оттока в офлайн (premortem Risk 5) |
| US-014 | Управление объявлениями: редактирование, архивация, восстановление своих/чужих (согласно ролевой модели) | Актуальность каталога предприятия — устаревшие позиции скрыты, новые добавлены |
| US-016 | Черновик: сохранение без публикации, видимость только автору и администратору | Подготовка объявлений без преждевременной публикации |
| US-017 | Жизненный цикл объявления: архивация, восстановление, снятие с публикации | Гибкое управление моментом появления объявления в поиске без потери данных |
