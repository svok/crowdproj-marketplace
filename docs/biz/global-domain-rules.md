# Global Domain Rules — CrowdProj Marketplace
**Связанные документы:** BR, Lean Canvas, Stakeholders
---
## 1. Доменные политики
Глобальные правила для всего продукта — утверждения формата «[Субъект] всегда/не может [действие]».
- **Запрет контактов в объявлении.** Размещение персональных контактных данных (телефон, email) в тексте объявления запрещено. Связь с контрагентом осуществляется через форму обратной связи на платформе (контактные данные предприятия не раскрываются публично в соответствии с 152-ФЗ). Публичная карточка предприятия (BR-005) содержит только: название компании, ИНН, сфера деятельности, регион.
- **Направление объявления неизменно.** Направление объявления (продажа ↔ закупка) не может быть изменено после публикации.
- **Enterprise — обе роли (динамическая).** Предприятие может выступать одновременно и закупщиком, и поставщиком. Роль является динамической — определяется наличием объявлений соответствующего направления (direction), а не атрибутом, выбранным при регистрации. Предприятие не указывает свою роль при создании профиля.
- **Согласие на ПД обязательно.** Согласие на обработку персональных данных (152-ФЗ) обязательно для всех представителей предприятия — как при регистрации предприятия (BR-001), так и при приглашении сотрудников (BR-002).
- **Блокировка только активных.** Блокировка применима только к активным объявлениям; черновики не блокируются.
- **Email-напоминание — одно.** Email-напоминание о новых матчингах — одно письмо за цикл неактивности, не дайджест.

## 2. Бизнес-ограничения
Ограничения уровня продукта — количественные или качественные лимиты. Конкретные значения задаются конфигурацией системы.
| Ограничение | Область действия | Комментарий |
|-------------|------------------|-------------|
| Минимальная длина поискового запроса | Поиск по предприятиям, объявлениям | Предотвращает нерелевантные пустые/короткие запросы. Значение задаётся конфигурацией. Базовое значение (согласно BR-005, BR-007) — 3 символа |
| Настраиваемый лимит пагинации | Списки объявлений, реестр предприятий | Списки должны быть постраничными. Лимит записей на страницу — настраиваемый параметр |
| Состав обязательных полей при регистрации | Регистрация предприятия | Часть полей (название компании, ИНН, email) — обязательные. Состав mandatory-полей определяется конфигурацией |
| SLA доступности | Платформа в целом | Доступность платформы: 99.5% (≈3.5 ч простоя/мес) — базовая конфигурация для MVP (согласовано со стейкхолдерами). Значение может быть пересмотрено в пост-MVP — задаётся конфигурацией |
| Таймаут поисковых запросов | Поиск, матчинг | Время отклика поисковых запросов ограничено. Конкретное значение задаётся конфигурацией |

## 3. Правила взаимодействия
Как акторы/сущности взаимодействуют на бизнес-уровне — «[Кто] может/не может [действие] с [чем]».
| Правило | Затрагиваемые субъекты | Условие |
|---------|------------------------|---------|
| Администратор может редактировать, архивировать и восстанавливать любые объявления предприятия. Сотрудник — только свои | Администратор предприятия, Сотрудник | На основании ролевой модели (BR-003) |
| При создании/обновлении объявления система показывает встречные объявления (продажа ↔ закупка) | Система, Закупщик, Поставщик | Механика «откликнуться» отсутствует в MVP (BR-004) |
| Пользователь не может зарегистрировать новое предприятие на email, уже привязанный к другому предприятию | Пользователь, Предприятие | При регистрации (BR-001) |
| Контактные данные контрагента — только через профиль организации. В тексте объявления размещать контакты запрещено | Закупщик, Поставщик | При создании/просмотре объявления (BR-003) |
| Направление объявления (продажа ↔ закупка) не может быть изменено после публикации | Предприятие, Объявление | После публикации (BR-003) |
| Владелец заблокированного объявления может его отредактировать, но не может опубликовать до разблокировки модератором | Предприятие, Модератор/Система | Для объявлений со статусом «заблокировано» (BR-008) |

## 4. Статусные политики
Глобальные правила статусов — только общие, не привязанные к конкретным сущностям. Полная матрица переходов между статусами — см. BR-008 (Матрица разрешённых переходов).
| Политика | Применяется к |
|----------|---------------|
| Черновики не видны другим пользователям | Сущности со статусной моделью (на текущий момент — объявления) в статусе «черновик» |
| Архивные сущности не участвуют в поиске и матчинге, доступны только для чтения | Сущности со статусной моделью (на текущий момент — объявления) в статусе «архив» |
| Заблокированные сущности не отображаются в публичных списках и поиске | Сущности с флагом блокировки |
| Блокировка применима только к активным сущностям. Черновики не блокируются | Активные сущности (на текущий момент — объявления) |
| Флаг блокировки сохраняется при изменении основного статуса. Перечень переходов — см. BR-008 | Сущности с флагом блокировки |
| Редактирование содержимого архивных сущностей недоступно — требуется восстановление | Сущности со статусом «архив» (на текущий момент — объявления) |
