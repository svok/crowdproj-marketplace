# BR-005: Реестр предприятий (список зарегистрированных компаний)

## Бизнес-требование
В соответствии с MVP Scope (P1: «Список предприятий (реестр всех зарегистрированных компаний на платформе)») система должна предоставлять авторизованным пользователям реестр всех предприятий с полнотекстовым поиском по всем публичным полям карточки (название компании, ИНН, сфера деятельности, регион) и фильтром по региону. Это необходимо для:
- **Закупщики** (Personas, CJM-закупщик этап 2) — находят поставщиков напрямую, даже если у тех ещё нет активных объявлений. Снижает зависимость от единственного поставщика (Lean Canvas, проблема №1).
- **Поставщики** (Personas, CJM-поставщик этап 2) — видят, какие закупщики уже на платформе, оценивают потенциальный рынок и могут инициировать контакты (Lean Canvas, проблема №2).
- **Митигация chicken-and-egg** (premortem Risk 1) — реестр создаёт видимость активности даже при малом количестве объявлений: пользователь видит список компаний и понимает, что платформа живая.
Реестр является P1-фичей MVP — не блокирует запуск, но существенно повышает ценность платформы для обеих сторон.

## User Stories
### US-1: Поиск поставщиков закупщиком через реестр
* **As a** закупщик (Инженер по закупкам)
* **I want** просматривать реестр предприятий и искать поставщиков по названию, сфере деятельности и региону
* **So that** найти альтернативных поставщиков без ручного сбора данных и обзвонов
### US-2: Оценка рынка поставщиком через реестр
* **As a** поставщик (производитель комплектующих)
* **I want** просматривать реестр предприятий и видеть, какие закупщики зарегистрированы на платформе
* **So that** оценить потенциальный рынок и понять, стоит ли инвестировать время в наполнение каталога

## Acceptance Criteria
### AC для US-1: Поиск поставщиков закупщиком через реестр
* **Given:** закупщик авторизован на платформе
* **When:** открывает раздел «Предприятия»
* **Then:** система отображает постраничный список всех зарегистрированных предприятий с публичными данными: название компании, ИНН, сфера деятельности (free-text), регион (справочник субъектов РФ)
* **Given:** закупщик на странице реестра предприятий
* **When:** вводит текст в строку поиска (не менее 3 символов)
* **Then:** система выполняет полнотекстовый поиск по всем публичным полям карточки предприятия (название компании, ИНН, сфера деятельности, регион) и обновляет список. Результаты отсортированы по релевантности
* **Given:** закупщик на странице реестра предприятий
* **When:** применяет один или несколько фильтров (регион + дополнительные фильтры с кастомной логикой комбинирования)
* **Then:** система отображает только предприятия, соответствующие всем выбранным фильтрам одновременно (AND-логика)
* **Given:** закупщик видит список предприятий в реестре
* **When:** нажимает на название предприятия
* **Then:** система открывает публичную карточку предприятия с названием компании, ИНН, сферой деятельности (free-text) и регионом (справочник субъектов РФ). Контактные данные (email, телефон) не отображаются
* **Given:** поиск или фильтрация не дали результатов
* **When:** список предприятий пуст
* **Then:** система показывает сообщение «По вашему запросу ничего не найдено» и предлагает изменить параметры поиска/фильтров
### AC для US-2: Оценка рынка поставщиком через реестр
* **Given:** поставщик авторизован на платформе и открыл раздел «Предприятия»
* **When:** просматривает список или открывает карточку предприятия
* **Then:** система показывает те же публичные данные, что и закупщику — название компании, ИНН, сфера деятельности (free-text), регион (справочник субъектов РФ). Контактные email/телефон не отображаются, согласно 152-ФЗ
* **Given:** на платформе зарегистрировано 2 и менее предприятий
* **When:** пользователь открывает реестр
* **Then:** система показывает сообщение «На платформе пока мало компаний — станьте первым!» с приглашением зарегистрировать своё предприятие (ссылка на BR-001: регистрация предприятия)

## Детали реализации (для проектирования)
### Сортировка списка
Система поддерживает переключение вариантов сортировки:
1. **По релевантности** — при активном поисковом запросе (результаты полнотекстового поиска, по убыванию relevance score)
2. **По названию компании** (А–Я)
3. **По дате регистрации** (от новых к старым)
4. **По дате регистрации** (от старых к новым)
При отсутствии поискового запроса сортировка по умолчанию — по дате регистрации (от новых к старым).
### Поиск
- **Минимальная длина запроса:** 3 символа (при вводе <3 символов поиск не выполняется, показывается подсказка «Минимум 3 символа»)
- **Область поиска:** все публичные поля карточки предприятия (название компании, ИНН, сфера деятельности, регион)
- **Тип:** полнотекстовый поиск (полное или частичное совпадение)
### Фильтры
- **Фильтр по региону:** справочник субъектов РФ (85 регионов), одиночный выбор
- **Дополнительные фильтры** (если появятся в будущем) — комбинируются с существующими по AND-логике
- Архитектура фильтрации должна предусматривать расширение набора фильтров (кастомная логика комбинирования)
### Логирование
- Все действия в реестре (открытие раздела «Предприятия», поиск, применение фильтров, просмотр карточки предприятия) логируются как стандартные события
- Аналитика востребованности строится исключительно на основе логов — отдельные кастомные счётчики НЕ предусмотрены
- Формат логов: событие + timestamp + session_id (без User ID для соблюдения 152-ФЗ, если событие не требует аутентифицированной сессии)
### Справочники
- **Регион:** справочник субъектов РФ (85 регионов, согласно Конституции РФ). Значение выбирается из выпадающего списка при регистрации предприятия (BR-001)
- **Сфера деятельности:** свободное текстовое поле (free-text) без справочника. Пользователь вводит произвольный текст при регистрации (BR-001)
