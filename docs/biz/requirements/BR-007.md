# BR-007: Список объявлений предприятия и глобальный поиск по объявлениям
**Связанные документы:** MVP Scope, Lean Canvas, Personas, stakeholders.md, BR-003, BR-004, BR-005
---
## Стейкхолдерские ограничения
| Стейкхолдер | Ограничение / NFR | Влияние на BR |
|-------------|-------------------|---------------|
| Закупщик, Поставщик | Доступность платформы 99.5% (SLA) | Страницы списков объявлений и глобальный поиск должны быть доступны 24/7 — отказ означает, что пользователи не видят предложения контрагентов |
| Закупщик, Поставщик | Производительность поиска <5 секунд на 1 000 объявлений (MVP); целевой <2 секунд на 10 000 (пост-MVP) | Поиск по объявлениям (глобальный и внутри предприятия) должен выполняться не дольше 5 секунд при объёме до 1 000 объявлений |
| Команда (операционная безопасность) | Логирование ключевых действий с MVP | Все действия в списках объявлений (поиск, фильтрация, просмотр карточки) должны логироваться для аудита и аналитики |
| Регуляторы (Роскомнадзор) | 152-ФЗ: согласие на обработку ПД, хранение на территории РФ | Персональные контактные данные сотрудников (email, телефон) не размещаются в тексте объявления — для связи используется профиль организации. В карточке объявления обязательно отображается ссылка на профиль предприятия-владельца, где доступны публичные данные организации (название, ИНН, сфера деятельности, регион) |

## Бизнес-требование
В соответствии с MVP Scope (P1: «Список объявлений предприятия (все активные объявления конкретной компании в одном месте)») система должна предоставлять авторизованным пользователям единый интерфейс для просмотра и поиска объявлений — своих (управление) и чужих (просмотр из карточки предприятия, реестра и через глобальный поиск). Это необходимо для:
- **Закупщики** (Personas, CJM-закупщик этап 2) — оценивают ассортимент поставщиков без ручного запроса, сравнивают предложения, принимают решение о контакте. Снижает зависимость от единственного поставщика (Lean Canvas, проблема №1).
- **Поставщики и Менеджеры по продажам** (Personas) — управляют каталогом своих объявлений, мониторят рынок и видят предложения конкурентов для корректировки ценовой политики и ассортимента (Lean Canvas, проблема №2).
- **Митигация chicken-and-egg** (premortem Risk 1) — глобальный поиск по всем объявлениям создаёт видимость наполнения платформы: даже при малом числе предприятий пользователь видит десятки объявлений.
Без списка объявлений предприятия карточка предприятия (BR-005) — просто визитка без информации о товарах. Без глобального поиска пользователи не могут целенаправленно искать конкретные товары, полагаясь только на матчинг при создании объявления (BR-004).

## User Stories
### US-1: Просмотр объявлений поставщика из реестра предприятий
* **As a** закупщик (Инженер по закупкам)
* **I want** из карточки предприятия в реестре перейти к списку всех его объявлений
* **So that** оценить ассортимент и выбрать подходящие позиции без отдельного запроса поставщику
### US-2: Управление каталогом своих объявлений
* **As a** представитель предприятия (администратор или сотрудник)
* **I want** видеть все свои объявления в одном месте с фильтрацией по статусу и направлению
* **So that** управлять каталогом, контролировать актуальность и быстро находить нужные позиции
### US-3: Мониторинг объявлений других предприятий
* **As a** менеджер по продажам (сбыт предприятия)
* **I want** просматривать объявления других предприятий (конкурентов и закупщиков)
* **So that** анализировать рыночные предложения, мониторить цены и ассортимент конкурентов
### US-4: Глобальный поиск по всем объявлениям
* **As a** представитель предприятия (любая роль)
* **I want** искать по всем объявлениям на платформе по названию, категории и другим параметрам
* **So that** находить нужные товары или заявки, не дожидаясь матчинга при создании объявления

## Acceptance Criteria
### AC для US-1: Просмотр объявлений предприятия из реестра
* **Given:** пользователь открыл публичную карточку предприятия (из реестра BR-005)
* **When:** нажимает «Объявления предприятия»
* **Then:** система отображает постраничный список всех активных объявлений данного предприятия с названием, категорией, direction (продажа/закупка) и датой публикации
* **Given:** пользователь просматривает список объявлений предприятия
* **When:** нажимает на любое объявление
* **Then:** система открывает карточку объявления с публичными данными о товаре (название, описание, динамические атрибуты категории). В карточке обязательно отображается ссылка на профиль предприятия-владельца (аналогично BR-005), где доступны публичные данные организации. Кнопки редактирования/архивации отсутствуют — это чужие объявления
* **Given:** у предприятия нет активных объявлений
* **When:** пользователь открывает раздел «Объявления предприятия»
* **Then:** система показывает сообщение «У предприятия пока нет активных объявлений» и предлагает вернуться к реестру предприятий
### AC для US-2: Управление каталогом своих объявлений
* **Given:** представитель предприятия авторизован и открыл раздел «Мои объявления»
* **When:** страница загружена
* **Then:** система отображает все объявления своего предприятия с фильтрацией по:
  - статусу (черновик / активные / архив / заблокировано / все)
  - direction (продажа / закупка / все)
  - поиску по названию или категории (мин. 3 символа)
* **Given:** представитель предприятия на странице «Мои объявления» и выбрал фильтр
* **When:** применяет один или несколько фильтров
* **Then:** система обновляет список согласно AND-логике фильтров
* **Given:** представитель предприятия на странице «Мои объявления»
* **When:** нажимает на своё активное объявление
* **Then:** система открывает карточку объявления с возможностью редактирования/архивации (согласно ролевой модели BR-003 — администратор может редактировать любые объявления предприятия, сотрудник — только свои)
* **Given:** представитель предприятия просматривает список своих объявлений и выполняет поиск
* **When:** поиск не дал результатов
* **Then:** система показывает сообщение «По вашему запросу ничего не найдено» и предлагает изменить параметры поиска/фильтров
### AC для US-3: Мониторинг объявлений других предприятий
* **Given:** авторизованный пользователь открыл карточку любого предприятия из реестра
* **When:** просматривает список его объявлений
* **Then:** система отображает публичные данные каждого объявления: название, категория, direction, описание, динамические атрибуты (согласно категории), дата публикации. Персональные контактные данные в тексте объявления не допускаются — для связи с предприятием используется ссылка на профиль организации
* **Given:** пользователь просматривает список объявлений другого предприятия
* **When:** нажимает на название предприятия-владельца
* **Then:** система открывает публичную карточку этого предприятия (из BR-005)
### AC для US-4: Глобальный поиск по всем объявлениям
* **Given:** авторизованный пользователь открыл раздел «Все объявления»
* **When:** вводит поисковый запрос (не менее 3 символов)
* **Then:** система выполняет полнотекстовый поиск по названиям и описаниям всех активных объявлений всех предприятий. Результаты отсортированы по релевантности
* **Given:** пользователь на странице «Все объявления»
* **When:** применяет один или несколько фильтров (категория, direction, регион предприятия)
* **Then:** система фильтрует объявления по AND-логике. Регион соответствует справочнику субъектов РФ (85 регионов, как в BR-005 и BR-001)
* **Given:** пользователь просматривает результаты глобального поиска
* **When:** нажимает на объявление
* **Then:** система открывает карточку объявления с информацией о товаре, названием предприятия-владельца и ссылкой на профиль организации (аналогично BR-005), где доступны публичные данные предприятия
* **Given:** глобальный поиск не дал результатов
* **When:** список пуст
* **Then:** система показывает сообщение «По вашему запросу ничего не найдено» и предлагает изменить параметры поиска/фильтров
* **Given:** пользователь выполняет поиск, просмотр или фильтрацию в разделе «Все объявления» или в списке объявлений предприятия
* **When:** операция завершена
* **Then:** система создаёт запись в логе аудита с указанием: пользователь (ID), тип операции (поиск/просмотр/фильтрация), контекст (глобальный/по предприятию), параметры запроса, количество результатов, дата/время

## Детали реализации (для проектирования)
### Страницы и навигация
1. **Список объявлений предприятия** — доступен из карточки предприятия (BR-005) по ссылке «Объявления предприятия»
2. **Мои объявления** — раздел в личном кабинете представителя предприятия
3. **Глобальный поиск (Все объявления)** — отдельный раздел навигации платформы, доступный всем авторизованным пользователям
### Поиск
- **Минимальная длина запроса:** 3 символа (при вводе <3 символов поиск не выполняется, показывается подсказка)
- **Область поиска (глобальный):** названия и описания всех активных объявлений
- **Область поиска (свои):** названия и описания объявлений своего предприятия (активные + архив)
- **Тип:** полнотекстовый поиск (полное или частичное совпадение)
### Фильтры
- **Категория товара:** иерархическое дерево категорий (из BR-003), поддержка выбора подкатегорий
- **Direction:** продажа / закупка / все
- **Регион предприятия:** справочник субъектов РФ (85 регионов)
- **Статус (для своих):** активные / архив / все
- **AND-логика** при комбинировании нескольких фильтров
### Доступ без авторизации
Разделы «Все объявления» и список объявлений предприятия доступны только авторизованным пользователям. Неавторизованные пользователи перенаправляются на страницу входа.
### Сортировка по экранам
| Экран | Сортировка по умолчанию | Доступные варианты |
|-------|------------------------|---------------------|
| Список объявлений предприятия (US-1, US-3) | По дате публикации (от новых к старым) | По дате (новые/старые), по названию (А–Я) |
| Мои объявления (US-2) | По дате публикации (от новых к старым) | По дате (новые/старые), по названию (А–Я), по статусу |
| Глобальный поиск (US-4) | По релевантности (при поисковом запросе); по дате публикации (без запроса) | По релевантности, по дате (новые/старые), по названию (А–Я) |
### Пагинация
- **Лимит:** 20 записей на страницу
- **Тип:** постраничная (номера страниц + кнопки «Вперёд»/«Назад»)
- **Отображение:** номер текущей страницы / общее количество страниц, общее количество результатов
### Модель региона для объявлений
Регион для фильтрации объявлений наследуется от предприятия-владельца объявления (поле «регион» из BR-001). Объявление не имеет собственного региона — при создании объявления регион не запрашивается. Фильтр «регион» в глобальном поиске применяется к региону предприятия-владельца.
### Соотношение с матчингом (BR-004)
Глобальный поиск по объявлениям (BR-007) и матчинг (BR-004) — независимые механики:
- **Матчинг (BR-004):** событийный, срабатывает при создании/обновлении объявления. Показывает встречные объявления (продажа ↔ закупка) в контексте конкретного объявления.
- **Глобальный поиск (BR-007):** пользовательский, инициируется пользователем. Показывает все объявления по запросу, без привязки к direction и без привязки к конкретному объявлению пользователя.
