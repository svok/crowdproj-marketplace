# Customer Journey Map — CrowdProj Marketplace
## Пользователь
- **Персона:** Закупщик (Инженер по закупкам)
- **Роль в контексте предприятия:** Закупщик действует в составе промышленного предприятия, которое само является производителем продукции. Это же предприятие выступает и поставщиком для других заказчиков — но в данном CJM рассматривается только роль закупщика.
- **Сценарий:** Срочный поиск альтернативного поставщика комплектующих. Основной поставщик задерживает отгрузку фланцев по ГОСТу на 3 недели. Закупщику нужно найти альтернативу за 2–3 дня, чтобы избежать простоя производства.

## Этапы пути
| Этап | Действие | Мысли | Эмоции | Боли | Возможности |
|------|----------|-------|--------|------|-------------|
| **1. Получение заявки от цеха** | Получает от мастера цеха заявку с чертежом/ТЗ/ГОСТом на деталь. Выясняет, что основной поставщик задерживает поставку на 3 недели | «Где найти эту деталь? Сколько времени это займёт? Не сорвём ли производство?» | Тревога, раздражение, давление | Сжатые сроки, страх срыва производства, давление от руководства | CrowdProj может сократить поиск с недель до часов |
| **2. Поиск детали / поставщика по техдокументации** | Ищет в Google, Avito, Пульс цен, обзванивает знакомых поставщиков. Создаёт заявку на CrowdProj с прикреплённым ТЗ | «Везде информация разная, неполная. На CrowdProj нашло 3 варианта за 5 минут… неужели так бывает?» | Разочарование от хаоса → интерес к платформе | Разрозненность данных, нет единого окна, информация устаревшая, тратит 1–3 недели | AI-матчинг по техдокументации, верифицированные поставщики в одном месте |
| **3. Сравнение вариантов** | Сравнивает цены, сроки, рейтинг, отзывы о поставщиках. Отбирает 2–3 кандидата для пробного заказа | «Эти дешевле, но те быстрее. Кому можно доверять? А если качество не устроит?» | Сомнение, надежда | Нет объективных данных о качестве поставщиков, риск ошибиться с выбором | Premium-аналитика: рейтинг, история сделок, бенчмарки для обоснования выбора |
| **4. Пробный заказ (малая партия)** | Отправляет ТЗ выбранным поставщикам, получает КП, заказывает малую партию для проверки | «Проверим их в деле. Если качество устроит — перейдём на контракт. | **5. Оценка результатов пробной поставки** | Получает пробную партию, проверяет качество по ТЗ/ГОСТу. Оценивает: сроки соблюдены, качество соответствует | «Поставщик справился. Качество хорошее, сроки выдержали. Можно переходить к контракту.» | Удовлетворение, доверие | Нет инструмента для фиксации оценки поставщика в системе | Встроенный рейтинг поставщиков, история поставок |
| **6. Заключение долгосрочного контракта** | Согласовывает с юристами рамочный контракт, график поставок, цены. Заключает договор на регулярные поставки | «Наконец-то нормальные отношения. Теперь не надо каждый раз искать и рисковать.» | Уверенность, спокойствие | Юридическая бумажная волокита, долгое согласование, нет шаблонов | Цифровые контракты на платформе (post-MVP), шаблоны по 223-ФЗ |
| **7. Исполнение регулярных поставок** | Получает отгрузки по графику, проводит приёмку, подписывает УПД и акты | «Работает как часы. Хорошо, что есть надёжный канал. Не надо больше суетиться.» | Спокойствие, рутина | Часть документооборота всё ещё вручную, нет автоматических заявок | Автозаявки по графику, интеграция с ERP, электронный документооборот |
| **8. Продление / реновация контракта** | Оценивает результаты за период, решает продлить контракт или объявить новый тендер. Делится опытом с коллегами | «Может, ещё поставщиков посмотреть на платформе? Или этому доверять и продлить?» | Осознанность, лёгкое беспокойство | Нет агрегированной аналитики поставок за период, чтобы обосновать решение | Дашборд эффективности поставщиков, рейтинг по категориям |

## Ключевые инсайты
1. **Поиск детали по ТЗ — самый болезненный этап.** Закупщик тратит 1–3 недели на поиск через разрозненные источники. MVP (доска объявлений) даёт первичную ценность сводя заявки и предложения в одном окне; AI-матчинг по техдокументации (в последующих версиях) усилит сокращение времени до часов.
2. **Пробный заказ — критический барьер конверсии.** Без возможности быстро оформить пробную закупку заказчик не перейдёт к долгосрочному контракту. Платформа должна обеспечить простой инструмент «заказать образец/пробную партию».
3. **Документооборот — причина оттока.** На этапах 4–7 заказчик вынужден выходить из платформы для оформления документов. Чем дольше эти этапы вне платформы, тем выше риск, что и следующие сделки пройдут мимо CrowdProj.

## Рекомендации
| Проблема | Решение | Приоритет |
|----------|---------|-----------|
| Поиск детали занимает 1–3 недели из-за разрозненных источников | Доска объявлений + ручной сбор поставщиков (MVP); AI-матчинг по ТЗ — в последующих версиях | Medium |
| Нет доверия к незнакомым поставщикам на этапе сравнения | Верификация поставщиков + рейтинг + история сделок | High |
| Невозможно обосновать выбор поставщика перед руководством | Premium-аналитика: бенчмарки цен, рейтинг, история | Medium |
| Бумажная волокита на этапе контрактации | Шаблоны контрактов + интеграция с 1С/ERP | Medium |
| Нет инструмента для автоматических повторных закупок | Автозаявки по графику + автопродление контрактов | Low |
| Поиск альтернатив вручную при срыве поставок | Умные уведомления: «Ваш поставщик задерживает отгрузку — показать альтернативы» | Low |
