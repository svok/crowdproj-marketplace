# Business Viewpoint: user
**Связанные документы:** Domain Model, User Scenarios, Personas, BR-002
---
## Business Meaning
User — это учётная запись представителя предприятия на платформе CrowdProj Marketplace. Позволяет сотрудникам зарегистрированных компаний (закупщикам, поставщикам, менеджерам по продажам) работать от имени организации: размещать объявления, искать контрагентов, участвовать в матчинге. Одно предприятие может иметь несколько user'ов с разными ролями (администратор, сотрудник), что обеспечивает многопользовательский режим работы внутри одной компании.
В сервисе parties user хранится как кэш данных из внешнего IdS — parties не управляет жизненным циклом пользователя, а только отражает его актуальное состояние для отображения в UI.

## Lifecycle Expectations
*Примечание: parties хранит только конечный статус кэша из IdS (active/inactive). Полный жизненный цикл пользователя (приглашён → активен → отключён) управляется в IdS. Ниже — бизнес-смысл этапов с точки зрения платформы.*
| Этап (бизнес-состояние) | Бизнес-смысл | Ожидаемая длительность |
|-------------------------|-------------|----------------------|
| Приглашён | Администратор отправил приглашение по email, сотрудник ещё не активировал учётную запись. Не может работать на платформе от имени предприятия | до истечения TTL ссылки-приглашения |
| Активен | Сотрудник установил пароль и работает от имени предприятия. Виден в списке сотрудников предприятия, может размещать объявления и участвовать в матчинге | неопределённо — пока администратор не отключит или IdS не деактивирует |
| Отключён | Доступ сотрудника отозван администратором. Не может входить на платформу от имени предприятия. Учётная запись сохраняется для возможности восстановления | постоянно — до повторного приглашения администратором |
| Кэш неактивен (inactive) | Пользователь деактивирован/удалён в IdS (уволен). Не отображается в публичных UI-компонентах (карточки товаров, профили предприятий) | постоянно — пока IdS не вернёт статус active |

## Visibility Semantics
По умолчанию профиль сотрудника виден только другим сотрудникам того же предприятия (внутренняя видимость). Пользователь или администратор могут сделать профиль публичным — атрибут публичности хранится в IdS, parties кэширует его как часть данных пользователя.
parties/user хранит только статус кэша `active`/`inactive`. Детальная гранулярность статусов (приглашён, активен, отключён) — ответственность IdS. Admin UI получает полные статусы напрямую из IdS, parties предоставляет только базовый статус видимости для публичных компонентов.
| Актор | Видит user'ов? | Условие |
|-------|---------------|---------|
| Сам user (владелец учётной записи) | Да | Всегда — свой профиль |
| Администратор предприятия | Да | Сотрудников своего предприятия (через связь PartyId). Полные статусы (приглашён/активен/отключён) — из IdS напрямую |
| Сотрудник предприятия | Да | Других сотрудников своего предприятия (активных — через parties) |
| Внешние пользователи (другие предприятия / незарегистрированные) | Да | Только если атрибут публичности профиля в IdS установлен в true |
| Система (публичные UI-компоненты) | Да | Только user'ы со статусом кэша active — для отображения DisplayName в карточках товаров и профилях предприятий |

## Ownership Semantics
| Владелец (бизнес-роль) | Тип владения | Может передать? |
|------------------------|-------------|-----------------|
| Предприятие (через администратора) | Полное — учётная запись сотрудника принадлежит предприятию, не физическому лицу. Администратор управляет созданием, доступом и отключением | Да — администратор может передать управление другому администратору предприятия |
| Физическое лицо (сотрудник) | Частичное — владеет своими персональными данными (email, DisplayName) согласно 152-ФЗ. Может отозвать согласие на обработку ПД | Нет — не может передать учётную запись другому лицу. При увольнении учётная запись отключается администратором |
