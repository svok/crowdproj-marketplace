# Screen: Карточка предприятия

**Entity:** party  
**Operation:** RegisterParty  
**Связанные документы:** [screen-flows](screen-flows-RegisterParty.md), [domain-model](../domain-model.md), [use-case](../use-cases/uc-party-RegisterParty.md)

---

## Назначение экрана

Пользователь просматривает детальную информацию о зарегистрированном предприятии: название, ИНН, статус, контактные данные для связи, сферу деятельности и регион. С карточки можно перейти к списку объявлений предприятия или вернуться в реестр.

---

## Поля экрана

Только то что пользователь реально видит и с чем взаимодействует.

| Поле | Тип (текст / ввод / выбор / toggle) | Источник (поле из domain-model) | Обязательность | Только чтение? |
|------|-------------------------------------|--------------------------------|----------------|----------------|
| Название предприятия (PartyName) | текст | PartyName (mandatory) | Да | Да |
| ИНН (Inn) | текст | Inn (mandatory) | Да | Да |
| Статус (CacheStatus) | бейдж | CacheStatus (mandatory) | Да | Да |
| Тип регистрации (RegistrationType) | текст | RegistrationType (mandatory) | Да | Да |
| Сфера деятельности (BusinessSphere) | текст | BusinessSphere (DynamicAttributes) | Нет | Да |
| Регион (Region) | текст | Region (DynamicAttributes) | Нет | Да |
| Контактный email (ContactEmail) | текст (ссылка) | ContactEmail (DynamicAttributes) | Нет | Да |
| Контактный телефон (ContactPhone) | текст (ссылка) | ContactPhone (DynamicAttributes) | Нет | Да |

> **Примечание о видимости контактов:** ContactEmail и ContactPhone — **контакты организации** (юридического лица) для связи по закупкам/продажам. Это публичная бизнес-информация, не относящаяся к персональным данным физических лиц (152-ФЗ). Отображаются всем пользователям, просматривающим карточку предприятия. Предположение в screen-flows-RegisterParty.md о видимости «только для администратора» уточнено — оно относится к административным контактам сотрудников, а не к бизнес-контактам организации.
| Дата создания (createdAt) | текст (дата) | createdAt | Да | Да |

> **ОГРН (Ogrn):** Поле присутствует в domain-model (DynamicAttributes) и screen-flows, но **не отображается на экране карточки предприятия**. ОГРН не является контактной информацией для связи по закупкам/продажам — не публикуется в публичных карточках согласно BR-005. Отображение ОГРН (при необходимости) — ответственность административной панели предприятия, не публичной карточки.

### Статусные бейджи (CacheStatus)

| Значение CacheStatus | Отображение |
|----------------------|-------------|
| active | Зелёный бейдж «Активно» — организация активна в IdS |
| inactive | Серый бейдж «Неактивно» — организация отключена/заблокирована/удалена в IdS |

### Тип регистрации (RegistrationType)

| Значение RegistrationType | Отображение |
|--------------------------|-------------|
| real | «Зарегистрировано в IdS» (MVP — всегда `real`) |
| auto | «Автоматически создано» (MVP+) |

## Действия

Кнопки, жесты, переходы которые пользователь может совершить на этом экране.

| Действие | Триггер | Что происходит | Куда ведёт (следующий экран) |
|----------|---------|---------------|------------------------------|

| Просмотреть объявления предприятия | Кнопка «Объявления предприятия» | Переход на страницу со списком объявлений данной организации | Список объявлений предприятия |
| Связаться по email | Клик по email (ссылка) | Открытие почтового клиента по умолчанию | Внешнее приложение |
| Позвонить/написать по телефону | Клик по телефону (ссылка) | Открытие набора номера или мессенджера | Внешнее приложение |

## Валидация

### Front-end (до отправки запроса)

Карточка предприятия — экран только для чтения. Front-end валидация не требуется.

| Проверка | Сообщение пользователю |
|----------|----------------------|
| — | — |

### Back-end (после отправки запроса)

Ошибки, которые приходят с сервера при загрузке данных карточки предприятия.

| Ошибка | Сообщение | Где показать |
|--------|-----------|-------------|
| Предприятие не найдено (404) | «Предприятие не найдено. Возможно, оно было удалено или отключено.» | Полноэкранная страница + кнопка «Вернуться в реестр» |
| Ошибка сервера (500) | «Не удалось загрузить информацию о предприятии. Попробуйте снова.» | Alert в верхней части экрана |

> Ошибки async-создания Party (RegisterParty) не отображаются на этом экране — если организация не создана, карточка просто недоступна (404). Ошибки outbox (party создан, событие не опубликовано) не видимы пользователю — Party отображается корректно.

## Пустые состояния

| Состояние | Что видит пользователь | Действие |
|-----------|----------------------|----------|
| Loading (загрузка карточки) | Скелетон-заглушки блоков информации (название, ИНН, поля) + спиннер | Автоматически — система загружает данные |
| Not Found (предприятие не найдено) | Полноэкранная страница: «Предприятие не найдено. Возможно, оно было удалено или отключено.» | Кнопка «Вернуться в реестр» — возврат на Экран 1 |
| Error (ошибка загрузки) | Alert в верхней части: «Не удалось загрузить информацию о предприятии. Попробуйте снова.» | Кнопка «Повторить» — повторная загрузка карточки |

> Empty-состояние (пусто) не применимо для карточки одного предприятия. Если предприятие не существует — отображается Not Found.
