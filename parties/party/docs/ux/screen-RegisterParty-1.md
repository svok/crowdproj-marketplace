# Screen: Реестр предприятий

**Entity:** party  
**Operation:** RegisterParty  
**Связанные документы:** [screen-flows](screen-flows-RegisterParty.md), [domain-model](../domain-model.md), [use-case](../use-cases/uc-party-RegisterParty.md)

---

## Назначение экрана

Пользователь просматривает реестр всех зарегистрированных предприятий, осуществляет полнотекстовый поиск, фильтрует по региону и сфере деятельности, сортирует результаты и переходит к карточке выбранного предприятия для просмотра детальной информации.

---

## Действия

Кнопки, жесты, переходы которые пользователь может совершить на этом экране.

| Действие | Триггер | Что происходит | Куда ведёт (следующий экран) |
|----------|---------|---------------|------------------------------|
| Открыть карточку предприятия | Клик по строке списка | Переход на детальную карточку предприятия | Экран 2 (Карточка предприятия) |
| Поиск по предприятиям (полнотекстовый) | Ввод текста (минимум <!-- parameters: domain.party.search_min_chars --> 3 символа) + Enter / автоматически | Список фильтруется по введённой строке | Остаётся на экране |
| Сброс поиска | Кнопка «✕» в строке поиска | Очистка строки поиска, список возвращается к полному реестру | Остаётся на экране |
| Пагинация | Клик по номеру страницы / стрелки «вперёд»/«назад» | Загрузка следующей страницы списка | Остаётся на экране |
| Фильтр по региону | Выбор из выпадающего списка | Список фильтруется по выбранному региону | Остаётся на экране |
| Фильтр по сфере деятельности | Выбор из выпадающего списка | Список фильтруется по выбранной сфере | Остаётся на экране |
| Сортировка | Клик по заголовку колонки / выбор из выпадающего меню | Переключение режима сортировки списка | Остаётся на экране |
| Сброс фильтров | Кнопка «Сбросить» (активна только при активных фильтрах или поиске) | Очистка всех фильтров (регион, сфера), поисковой строки и сброс сортировки на умолчание | Остаётся на экране |

## Сортировка

Режимы сортировки списка предприятий:

| Режим | Описание | Условие |
|-------|----------|---------|
| По релевантности | Предприятия, наиболее соответствующие поисковому запросу | Доступен только при активном поиске |
| По названию (А–Я) | Сортировка по PartyName в алфавитном порядке | Всегда доступен |
| По дате регистрации (сначала новые) | Предприятия, созданные недавно, в начале списка | Всегда доступен. **Режим по умолчанию** |
| По дате регистрации (сначала старые) | Предприятия, созданные давно, в начале списка | Всегда доступен |

## Валидация

### Front-end (до отправки запроса)

| Проверка | Сообщение пользователю |
|----------|----------------------|
| Поисковый запрос короче <!-- parameters: domain.party.search_min_chars --> 3 символов | «Введите минимум 3 символа для поиска» |
| Маска ввода для ИНН (только 10 или 12 цифр) | Подсказка при вводе: «ИНН: 10 цифр (юрлицо) или 12 цифр (ИП)» |

### Back-end (после отправки запроса)

Ошибки которые приходят с сервера. Поскольку RegisterParty — async-операция (событие из IdS), ошибки создания Party проявляются как отсутствие записи в реестре. Ошибки загрузки самого реестра — стандартные сетевые/серверные.

| Ошибка | Сообщение | Где показать |
|--------|-----------|-------------|
| Ошибка загрузки списка (сеть, сервер недоступен) | «Не удалось загрузить реестр предприятий. Проверьте соединение и попробуйте снова.» | Alert в верхней части экрана |
| Пустой реестр (нет ни одной организации) | «Организации не найдены» | Центральная область списка |
| Мало предприятий (1–2 на платформе) | «На платформе пока мало компаний — станьте первым!» | Баннер или сообщение над списком |
| Поиск/фильтры не дали результатов | «По вашему запросу ничего нет» | Центральная область списка |
| Party не появилась после регистрации (ошибка из IdS) | Не отображается явно — Party отсутствует в реестре. Пользователь видит, что организация не создана | Реестр без изменений |

## Пустые состояния

| Состояние | Что видит пользователь | Действие |
|-----------|----------------------|----------|
| Loading (загрузка реестра) | Скелетон-заглушки строк таблицы + спиннер в области заголовка | Автоматически — система загружает данные |
| Empty (реестр пуст, ни одной организации) | Сообщение «Организации не найдены» | Нет доступных действий на этом экране — реестр пуст. Регистрация нового предприятия выполняется через IdS (внешняя система) |
| Near-empty (1–2 предприятия на платформе) | Сообщение «На платформе пока мало компаний — станьте первым!» | Приглашение к регистрации нового предприятия. Регистрация выполняется через IdS (внешняя система) |
| No search results (поиск/фильтры не дали результатов) | Сообщение «По вашему запросу ничего нет» | Изменить поисковый запрос, сбросить фильтры кнопкой «Сбросить» |
| Error (ошибка загрузки) | Сообщение «Не удалось загрузить реестр предприятий. Проверьте соединение и попробуйте снова.» | Кнопка «Повторить» — повторная попытка загрузки |
