## Aggregate: category

Category — узел иерархического дерева категорий товаров. Категория определяет **шаблон атрибутов** (attribute_definitions) — какие характеристики товара поставщик заполняет в объявлении, их типы, обязательность и правила валидации. Шаблон наследуется и дополняется дочерними категориями (merge-семантика). Каждое объявление (ad) привязано к одной категории через `category_id`.

**Идентификатор:** CategoryId (UUID v4)
**Тип:** Aggregate Root
**Хранение:** state-sourced (Append-Only Ledger)

### Поля агрегата

| Поле | Тип | Обязательность | Описание |
|------|-----|----------------|----------|
| id | CategoryId (UUID v4) | Mandatory | Уникальный идентификатор категории |
| name | string | Mandatory | Название категории. Отображается в дереве категорий и в фильтрах. Не пустое |
| parent_id | CategoryId \| null | Optional | Ссылка на родительскую категорию. null = корневой узел дерева |
| attribute_definitions | AttributeDefinition[] \| null | Optional | Шаблон атрибутов товара для данной категории. Определяет, какие характеристики поставщик заполняет в объявлении: имя, тип, обязательность, правила валидации. null — наследуется от родителя (для non-root). Для корневой категории — Mandatory |
| sort_order | integer | Mandatory | Порядок сортировки среди sibling-узлов на одном уровне. >= 0 |
| status | CategoryStatus | Mandatory | Текущее состояние категории (draft / active / archived) |

---

### Шаблон атрибутов (attribute_definitions)

Поле `attribute_definitions` представляет собой **массив объектов**, каждый из которых описывает один атрибут товара. Массив определяет форму, которую заполняет поставщик при создании/редактировании объявления в данной категории. Имена атрибутов и их типы маппятся на поле `attributes[]` в агрегате ad.

#### Структура элемента attribute_definitions[]

| Поле | Тип | Обязательность | Описание |
|------|-----|----------------|----------|
| `name` | string | Mandatory | Название атрибута (уникальное в рамках шаблона категории). Отображается как label поля в форме объявления |
| `type` | enum {string, number, boolean} | Mandatory | Тип значения атрибута. Определяет формат ввода на UI (текстовое поле / числовой инпут / чекбокс) и тип поля value в ad.attributes[] |
| `required` | boolean | Mandatory | true — атрибут обязателен для заполнения при создании объявления; false — опционален |
| `rules` | ValidationRules | Optional | Правила валидации значения: min, max, regex, enum (список допустимых значений). null — без дополнительных правил |
| `unit` | string \| null | Optional | Эталонная единица измерения для числовых атрибутов (type=number). Используется для вычисления normalized_value в ad.attributes[]. Отображается рядом с полем ввода на UI |
| `placeholder` | string \| null | Optional | Текст-подсказка внутри поля ввода (placeholder в UI). Показывается, пока пользователь не ввёл значение. Пример: "Введите ГОСТ..." |
| `example` | string \| null | Optional | Пример заполнения атрибута. Отображается под полем ввода или в тултипе. Пример: "ГОСТ 12820-80" |
| `description` | string \| null | Optional | Развёрнутое описание атрибута — для UI (подсказка, что именно нужно указать). Отображается как help text под полем |

#### Правила валидации (ValidationRules)

| Поле | Тип | Описание |
|------|-----|----------|
| `min` | number \| null | Минимальное значение (для type=number) или минимальная длина строки (для type=string) |
| `max` | number \| null | Максимальное значение (для type=number) или максимальная длина строки (для type=string) |
| `regex` | string \| null | Регулярное выражение для валидации строки (для type=string) |
| `enum` | string[] \| null | Список допустимых значений (для type=string). Пример: ["Сталь 20", "Сталь 45", "Нержавейка"] |

#### Наследование шаблона

Дочерняя категория наследует `attribute_definitions` родительской и может дополнять/переопределять атрибуты:

1. Если `attribute_definitions = null` у дочерней категории — используется полный шаблон родителя (без изменений)
2. Если `attribute_definitions` задан — выполняется merge:
   - Новые атрибуты (name, отсутствующий в родительском шаблоне) — добавляются
   - Существующие атрибуты (name, присутствующий в родительском шаблоне) — переопределяются (полная замена объекта)
   - Удаление атрибута, определённого родителем, — **запрещено**
3. Конечный шаблон для категории = результат рекурсивного merge от корня до текущего узла

## Value Objects

| VO | Тип | Правила валидации |
|----|-----|-------------------|
| CategoryId | UUID v4 | Генерируется системой. Не пустой. Не изменяется после создания |
| CategoryStatus | enum {draft, active, archived} | Одно из трёх допустимых состояний |

Все остальные поля агрегата являются примитивами (string, UUID, integer) или встроенными структурами (массив AttributeDefinition[]).

## State Machine

### Состояния

| Состояние | Описание |
|-----------|----------|
| Draft | Категория создана, но не активна. Не видна в UI-дереве для выбора при создании объявления. Доступна для редактирования администратором |
| Active | Категория опубликована, видна в дереве категорий. Доступна для выбора при создании и редактировании объявлений. Участвует в поиске и фильтрации |
| Archived | Категория скрыта из дерева. Недоступна для выбора при создании новых объявлений. Существующие объявления в этой категории сохраняют статус, но категория помечается как устаревшая |

### Разрешённые переходы

| Из | В | Триггер (операция) | Guard (условие) |
|----|---|---------------------|-----------------|
| (создание) | Draft | CreateCategory | Администратор авторизован. name не пустой. parent_id — ссылка на существующую категорию или null. attribute_definitions — валидный массив или null (наследование). sort_order >= 0 |
| Draft | Active | ActivateCategory | attribute_definitions заполнен (либо свой массив, либо наследован от родителя — для non-root категорий). Для root-категории — собственный массив обязателен |
| Active | Archived | ArchiveCategory | Нет активных объявлений, привязанных к данной категории (согласно BR-003, BR-008) |
| Archived | Active | RestoreCategory | Категория существовала ранее и не была удалена |
| Active | Draft | DeactivateCategory | Нет активных объявлений, привязанных к данной категории |
| Draft | Archived | ArchiveCategory | Без дополнительных условий |

## Structural Invariants

Правила, истинные ВСЕГДА для агрегата category.

| Инвариант | Комментарий |
|-----------|-------------|
| `name` не пустой и не состоит только из пробелов | Название категории — осмысленный идентификатор |
| `parent_id` не может ссылаться на `id` этого же агрегата | Запрет самоссылки |
| В дереве категорий отсутствуют циклы (circular reference) | parent_id не может образовывать петлю при обходе вверх по дереву. Проверка при создании/изменении parent_id |
| `sort_order` >= 0 | Порядок сортировки — неотрицательное целое число |
| `attribute_definitions` дочерней категории — расширение (не сужение) шаблона родителя | Все атрибуты, определённые в родительском шаблоне, присутствуют в конечном шаблоне дочерней категории. Дочерняя может добавлять новые и переопределять существующие, но не удалять |
| Корневая категория (parent_id = null) имеет собственный `attribute_definitions` | Корень дерева не наследует шаблон, а определяет базу для всей ветки |
| `attribute_definitions` каждого элемента — уникальность `name` в рамках одного массива | Два атрибута с одинаковым именем в одной категории недопустимы |
| После `status=archived` изменение полей категории (name, attribute_definitions, parent_id) запрещено | Архивная категория доступна только для чтения. Для изменения требуется восстановление в active |
| `attribute_definitions` не может быть пустым массивом (`[]`) | Если атрибуты не нужны — null (наследование от родителя) или отсутствие атрибутов задаётся через отсутствие их в шаблоне |
| `name` элемента attribute_definitions уникален в рамках всего дерева категорий (опционально) | Глобальная уникальность имён атрибутов упрощает матчинг — атрибут «Диаметр» в разных ветках дерева — один и тот же атрибут |

## Domain Events

События, которые публикует агрегат category. Именование в Past Simple.

| Событие | Когда публикуется | Данные |
|---------|-------------------|--------|
| CategoryCreated | Категория создана (в статусе Draft) | categoryId, name, parentId, sortOrder, status |
| CategoryUpdated | Поля категории изменены (name, attribute_definitions, sort_order, parent_id) | categoryId, changedFields, updatedAt |
| CategoryActivated | Категория переведена из Draft в Active | categoryId, activatedAt |
| CategoryArchived | Категория переведена из Active/ Draft в Archived | categoryId, archivedAt |
| CategoryRestored | Категория восстановлена из Archived в Active | categoryId, restoredAt |
| CategoryDeactivated | Категория снята с публикации (Active → Draft) | categoryId, deactivatedAt |

### Стандартные поля каждого события

| Поле | Тип | Описание |
|------|-----|----------|
| event_id | UUID v4 | Уникальный идентификатор события |
| aggregate_id | CategoryId (UUID v4) | ID категории — источника события |
| event_type | string | Имя события (CategoryCreated, CategoryUpdated и т.д.) |
| occurred_at | timestamp (HLC) | Время возникновения события |
| idempotency_key | string | Ключ идемпотентности для deduplication |
| payload | JSONB | Данные события (специфичные для каждого типа) |
